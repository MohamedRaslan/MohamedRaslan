# Pytest QA Touch Plugin

## Summary

Pytest QA Touch Plugin connects automated Pytest execution with QA Touch test management so test results can move from local or CI runs into a shared reporting workflow.

## Problem

Automated tests often run in developer tools, while test management lives somewhere else. That gap makes it harder for QA and engineering teams to understand what ran, what failed, and where follow-up is needed.

## What It Shows

- Python package and Pytest plugin development.
- Test result collection and reporting workflow thinking.
- Practical quality engineering beyond only writing assertions.

## Quality Angle

This project is a good profile anchor because it shows a tool-builder mindset: improving the way teams use test results, not just creating more tests.
